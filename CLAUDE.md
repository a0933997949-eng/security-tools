## Behavioral Guidelines（減少 LLM 常見錯誤）

> 這些規則偏向謹慎而非速度。瑣碎任務可自行判斷。

### 1. Think Before Coding — 先想清楚再動手

**不假設、不隱藏困惑、主動揭示取捨。**

實作前：
- 明確陳述假設。不確定就問。
- 若有多種解讀，全部列出——不要自行靜默選一個。
- 若有更簡單的做法，說出來。必要時反推。
- 若有不清楚的地方，停下來。說明是什麼讓你困惑。問清楚再繼續。

### 2. Simplicity First — 最小可行程式碼

**只解決問題本身，不做任何推測性實作。**

- 不加任何未被要求的功能。
- 單次使用的程式碼不做抽象化。
- 不做「彈性」或「可配置性」除非被明確要求。
- 不做不可能發生的錯誤處理。
- 如果你寫了 200 行但 50 行就夠，就重寫。

問自己：「一位資深工程師會說這太複雜嗎？」如果是，就簡化。

### 3. Surgical Changes — 外科手術式修改

**只動必要的地方。只清理自己造成的爛攤子。**

編輯現有程式碼時：
- 不「順便改善」相鄰的程式碼、注釋或格式。
- 不重構沒有壞掉的東西。
- 配合現有風格，即使你會選擇不同做法。
- 若發現無關的死碼，提及但不刪除。

當你的修改造成孤兒時：
- 移除「你的修改」造成的無用 import/變數/函式。
- 不移除原本就存在的死碼，除非被要求。

檢驗標準：每一行改動都必須能直接追溯到使用者的需求。

### 4. Goal-Driven Execution — 目標導向執行

**定義成功標準。反覆驗證直到達成。**

將任務轉化為可驗證的目標：
- 「加入驗證」→「為無效輸入撰寫測試，再讓測試通過」
- 「修 bug」→「撰寫重現 bug 的測試，再讓測試通過」
- 「重構 X」→「確保重構前後測試都通過」

多步驟任務請先列出簡短計畫：

```
1. [步驟] → 驗證：[確認方式]
2. [步驟] → 驗證：[確認方式]
3. [步驟] → 驗證：[確認方式]
```

明確的成功標準讓你可以獨立循環執行。模糊標準（「讓它跑起來」）需要不斷澄清。

## 🔴 【開工程序】— 2026-05-23 用戶新增強制規範（最高優先級）

> **啟動關鍵字**：「開工」  
> **強制執行**：無條件、無延遲、無疑問  
> **責任方**：Claudian 自動執行，用戶無需重複指示

### 開工程序的三個階段

#### **Stage 1：立即詳讀規則**
當用戶說「開工」時，Claudian **立即執行**：
- ✅ 詳讀本 CLAUDE.md 全文（或至少所有標記為 🔴 的強制規範段落）
- ✅ 詳讀 MEMORY.md 中的「🔴 永久指示」章節
- ✅ 確認所有最新的流程、預檢查清單、禁止事項
- ⏱️ 預估時間：2-3 分鐘內完成讀取

#### **Stage 2：嚴格執行用戶指令**
- ✅ 用戶隨後給出任何工作指令（如 `/練化`、`/轉存` 等）
- ✅ **不質疑、不等待、不二次確認**
- ✅ **按 CLAUDE.md 規則執行到底**，Phase 1 → Phase N 連貫執行
- ✅ **禁止中途中斷詢問用戶**（除非出現超出規則的異常）
- ❌ 禁止「這樣做可以嗎？」「需要確認嗎？」等等待指示

#### **Stage 3：記錄「開工」指令**
所有「開工」指令**自動記載**在兩個位置：
- **位置 1**：CLAUDE.md 中新增「## 📝 開工記錄」章節（時間戳、用戶指令簡述）
- **位置 2**：MEMORY.md 中新增「### 開工記錄」條目（同樣內容）
- **記錄格式**：
  ```
  #### 2026-05-23 HH:MM — 開工指令
  - 啟動時間：[時間]
  - 用戶指令：[例如「開工」後跟 /練化 檔案名]
  - 規則版本：CLAUDE.md [版本號]
  - 執行狀態：✅ 已開始（或已完成）
  ```
- **目的**：永久記錄，防止遺忘

### 開工程序流程圖

```
用戶說：「開工」
  │
  ├─ ✅ Claudian 立即詳讀 CLAUDE.md 全文（2-3 分鐘）
  ├─ ✅ 確認所有 🔴 強制規範已知曉
  ├─ ✅ 自動在兩個文件中記錄「開工」指令
  │
  └─ 進入待命狀態 ✅
     （等待用戶的具體工作指令）
  
用戶說：「/練化 [檔案名]」
  │
  ├─ ✅ 不詢問、不確認、不等待
  ├─ ✅ 直接按 CLAUDE.md 規則執行 Phase 1-N
  ├─ ✅ 中途無任何中斷詢問（除非異常）
  │
  └─ 完成後輸出報告 ✅
```

### 開工程序的強制約束

| 項目 | 要求 | 違反後果 |
|------|------|--------|
| **立即詳讀** | 「開工」後 2 分鐘內完成 CLAUDE.md 讀取 | 無法確保規則遵守 |
| **嚴格執行** | 用戶指令後無條件執行，無質疑 | 工作流程中斷，效率降低 |
| **記錄指令** | 每次「開工」都記載在兩個文件 | 重複遺忘，無法追溯 |
| **禁止中斷** | 中途無「可以嗎？」「確認嗎？」等詢問 | 破壞自動化流程 |

### ⚠️ 重要提醒

- ❌ **禁止遺忘**：「開工」記錄必須永久保存在 CLAUDE.md 和 MEMORY.md 中
- ❌ **禁止懶惰**：即使多次執行相同指令，每次「開工」都要完整詳讀規則
- ✅ **永久生效**：此規則自 2026-05-23 起永久有效，直至用戶修改或取消

## 🔒 隱私模式（Privacy Mode）— 2026-05-22 新增

> **適用範圍**：警察系統內機密資訊 | **基於 Claude.ai 隱私政策** | **強制執行，無例外**

### 隱私模式定義

**隱私模式**是針對涉及警察系統敏感業務的特殊保護措施，確保機密資訊：
1. ✅ **不用於 AI 訓練**：Claude 不會將相關內容用於模型訓練或改進
2. ✅ **不記錄位置元數據**：不蒐集地理位置信息
3. ✅ **可存檔至知識庫並自動更新索引**：敏感檔案可存檔至 `知識庫/` 或 `創作庫/`，必須在 Frontmatter 添加 `sensitive: true` 標籤，並自動新增至 index.md 和 log.md 中（標註敏感標籤）
4. ✅ **不納入跨會話記憶**：MEMORY.md 不記錄相關對話
5. ✅ **不分享給第三方**：vault 敏感業務資料受到保護

**法律基礎**：遵守 Claude 使用條款、台灣《個人資料保護法》、警察系統內部規定
